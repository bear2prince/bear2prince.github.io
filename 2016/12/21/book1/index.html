Book1 Ch2

1.1 Object的属性可以动态声明

c.abc=3中给Object c动态声明了新的property abc

1.1.1 数组类型变量是指向数据的reference

b和a都是指向[1,2,3]的reference

1.1.1 Keng: 数组可作为Object使用

javascript中数组也能作为object使用。使用typeof函数查看数组的类型也是object。d.abc=3给数组d动态声明了property abc。但数组的property并不记入长度（length）。
此外，当给超出数组长度的元素赋值时，数组将直接扩展长度。同时将中间跨过的部分全部填充为undefined

1.2 Types

注意区分教材中提到的几种“type”： “built-in types”类比于其他语言中的“primitive types” “subtypes”指function和array是object的子类型 另外function还被称作“main type”，指的是 通过typeof函数查看函数的类型时显示的是function（而不是object）

1.3.1 Keng: NaN

对字符串（不是只含有数字的字符串）进行数字类型运算后，结果强制转换为数字类型，值为NaN。NaN没有大小，比较结果一定是false。

1.3.2 Truthy & Falsy values

关于各种类型空值是Truthy还是Falsy的快捷记忆方法： 除了object 及其 subtypes 空值 为 true 其余 types 空值都为 false

1.3.3 Keng: 强制转换带来的逻辑矛盾

[]==0值为true的原因是比较时[]被强制转换为数字类型， 值为0。然而如果将两者都转换为boolean类型（比如将[]或0直接用在if的条件中），结果却相反。
此外，空object{}强制转换为数字类型时，结果是NaN（不是0）
基于以上情况，比较时一定使用===。除数字类型的比较之外，其他类型的比较一定先用类型特有方法显示转化，再进行比较。避免隐式转化产生的错误无法定位。
具体转化规则参考文档:comparison coercion rules

1.3.4 字符比较大小

从测试结果看字符之间的大小比较按照ascii编码，而不是字母表（尚未经过文档确认）

2.1.2 对没有声明的变量赋值会产生全局变量

直接调用未声明的变量（b），会产生ReferenceError；而给未声明的变量赋值（a），在非strict mode下会生成全局变量。

2.1.2 Keng: 注意循环条件中的变量

for循环条件中的i没有声明，在非strict mode下直接被声明为全局变量，函数结束后依然存在。

2.1.2 var和let

var声明的变量的scope不受{}限制，所以变量a的scope包括了第一个函数内的全部范围，而在函数外没有scope。 let声明的变量的scope受{}限制，所以let声明的变量b只在括号内拥有scope。在括号外执行b=2时，产生了另一个全局变量b。

2.1.2 Keng: 重复声明

对同一变量i的声明只有第一个起作用。之后的声明被忽略（但赋值仍会执行），所以整个函数中的i都是同一个变量。

2.1.2 Keng: var和let不能对同一变量混用

单独使用let，两次声明均生效，循环条件和循环内容中的i不是同一变量； 单独使用var，后一次声明被忽略，循环条件和循环内容中的i是同一变量（小心无限循环） 当混用let和var时，var i = 1一句报错，重复声明。
