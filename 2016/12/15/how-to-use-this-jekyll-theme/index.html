chapter 1

foo1()的this是default binding，在非strict mode下，此this bind到global-object， this.bar1()调用global中的bar1函数（声明在global的变量或函数都是global-object的property）。bar1()的this是implicit binding，此this被bind到foo1()中this所指向的global-object。由于global-object中没有a，所以this.a为undefined。

chapter 2 2.1.

bar2()在foo2()中被plain and undecorated invoked，所以bar2()的this是default binding，指向global-object。由于global-object中没有a，所以this.a为undefined。

chapter 2 2.4.

new binding 新生成一个object，只有link prototype，没有内存拷贝，所以是空object，其中没有a。并且传入foo3的flag参数为false，所以foo3中并没有执行this.a = something，新object中没有a，所以输出为undefined。（当传入foo3的flag参数为true时，foo3执行this.a = something，因为new binding优先于implicit binding，此时this指向新创建的object，由于此object中没有a，所以创建a并赋值4。而obj3中a的值还是3）

Keng

当代码编译到function bar5()时并不会报错，因为函数体只在函数被调用时才执行（函数名在编译时就被分配）。 当代码运行到bar5()并执行函数体中foo5(obj)时也不会报错，因为babeljs将let翻译成var，obj的声明被hoisted，执行foo5(obj)时RHS可以找到obj，但当执行foo5()函数体时，由于obj还没有被赋值，执行fn.a（obj.a）时报错。

Keng

let 不允许函数、变量重名，在编译的时候就会报错。而var 允许函数、变量重名。

chapter 2 2.3.1.

写函数先确定输入输出、再写函数体，应用逻辑学迭代： - 首先确定bind函数的功能：得到一个将指定函数hard binding到指定object的函数 - 于是明确bind函数的输入参数为fn和obj，返回值为一个函数binded - 由于我们的目标是hard binding fn和obj，也就是说binded函数需要通过调用fn和obj来生成一个包含fn和obj的closure。由于我们的目标是绑定fn和obj，并且fn可以接收参数，确定关键字apply。binded函数体内容为fn.apply(obj, arguments)，此处arguments的类型为array。

chapter 2 3.

执行let bb_foo9 = bind(obj9, foo9)后，bb_foo9是binded()的reference，并且binded()生成一个包含obj9和foo9的closure。

执行let nb_foo9 = new bb_foo9(45)时，首先45作为参数传给binded()。由于binded()的closure，fn为foo9，obj为obj9，当执行fn.apply(obj, arguments)时，foo9的this指针指向obj9，将obj9.a赋值为45。

然后执行new，生成一个新的object并将binded()的this指针指向此object，最后将这个object赋值给nb_foo9。由于这个object中没有a，console.log(nb_foo9.a)输出undefined。

注意： - “this is a binding made for each function invocation”，当调用binded()时会生成一个binded()的this指针。 - new binding 优先级在 explicit binding 下不一定：new binding 不能overide closure 形式的 hard binding，new binding 的优先级高过(overide) bind()

chapter 2 3.

bind函数不但可以将this与对象（这里是null）绑定，同时也固定了一个参数（“p1”）。

chapter 2 4.2.

default binding时，binding的对象是global-object还是undefined 取决于“the strict mode status of the function content” 也就是被调用的函数在被定义时是否是strict mode（而非调用时）

Keng

调用foo2()时没有传入参数，自动补充undefined，返回a+b时因为+运算，强制转换为数字类型， 值为NaN，所以foo2()(324234,2)相当于NaN(324234,2)

chapter 2.3.1.

let a = bind(obj,foo)将closure中带有obj和foo的binded()函数地址赋给a。当执行a(3,4)时，a(3,4)返回foo.apply(obj,[3,4])的返回值。foo(a,b)的返回值为a+b，所以foo.apply(obj,[3,4])将返回7，console.log(a(3,4))输出7。 同理，console.log(b(5,6))输出-1。 此外，由于let a =bind(obj,foo)和let b =bind(obj,foo2)两处将函数foo()和foo2()的this指针都绑定到obj，所以两函数中的console.log(this.a)都将输出obj中的a，值为1。

chapter 2 2.4. 第四步return object

console.log(e)处输出为{c:1},foo()中this.c=1为e所指向的新object生成property c。由于e所指向的是一个对象，而非function，所以e(4,5)报TypeError错。 当函数返回值为object时，如foo1(),f所指向的object为foo1()返回的{d:this}。注意：当函数返回值为object时，new binding前三步按顺序执行，只在第四步时考虑return。
